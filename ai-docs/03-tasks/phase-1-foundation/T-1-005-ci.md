# T-1-005. CI: GitHub Actions — lint + test + build

> **Тип:** infra
> **Приоритет:** P1
> **Оценка:** 2 часа
> **Фаза:** 1
> **Статус:** blocked (отложено до prod-репо)

---

## Цель

Ни один PR не попадает в main без зелёного CI. Падающий CI = невозможность мёржа.

---

## Зависимости

- **Блокируется:** T-1-001, T-1-003 (pre-commit config)

---

## Что нужно сделать

1. Создать `.github/workflows/ci.yml`:
   ```yaml
   name: CI

   on:
     pull_request:
       branches: [main]
     push:
       branches: [main]

   concurrency:
     group: ${{ github.workflow }}-${{ github.ref }}
     cancel-in-progress: true

   jobs:
     lint:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: "3.12"
             cache: pip
         - run: pip install pre-commit
         - run: pre-commit run --all-files --show-diff-on-failure

     test:
       runs-on: ubuntu-latest
       services:
         postgres:
           image: postgres:16
           env:
             POSTGRES_DB: mstechnics_test
             POSTGRES_USER: postgres
             POSTGRES_PASSWORD: postgres
           options: >-
             --health-cmd pg_isready
             --health-interval 5s
             --health-timeout 5s
             --health-retries 10
           ports:
             - 5432:5432
         redis:
           image: redis:7
           options: >-
             --health-cmd "redis-cli ping"
             --health-interval 5s
           ports:
             - 6379:6379

       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: "3.12"
             cache: pip
         - run: pip install -e ".[dev,test]"
         - name: Run migrations
           env:
             DATABASE_URL: postgres://postgres:postgres@localhost:5432/mstechnics_test
             REDIS_URL: redis://localhost:6379/0
             DJANGO_SECRET_KEY: test-only
             DJANGO_DEBUG: "False"
             DJANGO_SETTINGS_MODULE: config.settings.test
           run: python manage.py migrate --noinput
         - name: Run tests
           env:
             DATABASE_URL: postgres://postgres:postgres@localhost:5432/mstechnics_test
             REDIS_URL: redis://localhost:6379/0
             DJANGO_SECRET_KEY: test-only
             DJANGO_DEBUG: "False"
             DJANGO_SETTINGS_MODULE: config.settings.test
           run: pytest --cov-report=xml
         - uses: codecov/codecov-action@v4
           with:
             files: ./coverage.xml
             fail_ci_if_error: false

     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - name: Build Docker image
           run: docker build -t mstechnics-web:ci .
         - name: Smoke test — run image
           run: |
             docker run --rm -e DJANGO_SETTINGS_MODULE=config.settings.dev \
               -e DJANGO_SECRET_KEY=x -e DATABASE_URL=sqlite:///:memory: \
               -e REDIS_URL=redis://localhost \
               mstechnics-web:ci python manage.py check
   ```

2. Создать `.github/workflows/security.yml` — раз в неделю:
   ```yaml
   name: Security
   on:
     schedule:
       - cron: "0 3 * * 1"  # Понедельник 3 утра
     workflow_dispatch:

   jobs:
     deps-audit:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with:
             python-version: "3.12"
         - run: pip install pip-audit
         - run: pip-audit -r requirements.lock
   ```

3. Добавить branch protection правила в GitHub (в UI, но задокументировать):
   - `main` protected
   - Required status checks: `lint`, `test`, `build`
   - Require PR reviews: 1
   - Require linear history

4. Добавить badge в README:
   ```markdown
   [![CI](https://github.com/OWNER/REPO/actions/workflows/ci.yml/badge.svg)](https://github.com/OWNER/REPO/actions/workflows/ci.yml)
   ```

---

## Критерии приёмки

- [ ] При открытии PR — запускаются jobs lint, test, build
- [ ] При успехе всех 3 — зелёная галочка, мёрж разрешён
- [ ] При падении — PR блокируется
- [ ] Тесты реально гоняются (не skip) — проверить локально `pytest`
- [ ] Coverage собирается
- [ ] Security scan работает по расписанию

---

## Что НЕ делать

- **НЕ деплой** в этой задаче — CI только проверка, деплой отдельно
- **НЕ добавляй** preview-environments — сложно, на Фазе 5
- **НЕ гоняй** E2E в CI в этой задаче — пока нет фронта

---

## Известные сложности

- На первом прогоне тесты скорее всего упадут, т.к. их пока нет. Если пусто — добавь плейсхолдер:
  ```python
  # tests/test_sanity.py
  def test_sanity():
      assert 1 + 1 == 2
  ```
  И `--cov-fail-under=0` временно, до Фазы 2 когда тесты реально появятся.
