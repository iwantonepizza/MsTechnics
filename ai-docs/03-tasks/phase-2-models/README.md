## Критические правила Фазы 2

1. **Миграции — forward-only.** Откат через новую миграцию, не через `--fake`.
2. **Каждая миграция тестируется на копии прод-БД** до мержа в main.
3. **Data-migrations не атомарны** — `atomic = False` + batch-processing.
4. **Удаление колонок/таблиц — отдельной миграцией через 2 недели** после переезда данных.
5. **string-FK сохраняются** в старых миграциях (не трогаем), обновляются только в моделях.
6. **Compat-shim обязателен** — старые импорты работают, пока идёт перенос.
