## Что нужно сделать

### Подход

Не делать отдельную «страницу для печати». Использовать `@media print` CSS поверх существующей карточки заявки — это правильный путь по доступности (распечатать можно прямо со страницы).

### Структура карточки в печати

Что обязательно на странице A4 (книжная):

1. **Шапка**: логотип «Суперсимметрия» (черно-белый, минимум 100 px) + текст «Бюро визуальных коммуникаций» + дата печати справа.
2. **Заявка №{id}** крупно, под шапкой.
3. **Контекст:** Экран, Город, Номер ячейки, Номер панели, Дата создания.
4. **Статус** (текущий).
5. **Комментарии по этапам** (мониторинг, контроль, сервис) — в виде timeline или таблицы.
6. **Исполнитель** (если назначен).
7. **Файлы/фото** — если есть, то thumbnail; ссылки на полные файлы (URL под thumbnail).
8. **Подвал**: место для подписи + дата подписи + «Распечатано {timestamp} из системы Суперсимметрия».

### CSS

В `frontend/src/pages/application-detail/ApplicationDetailPage.tsx` (или его CSS-module):

```css
@media print {
  /* скрываем глобальные шапки/сайдбары/футер/SSE indicator */
  .app-header, .app-sidebar, .footer, .sse-indicator, .toolbar,
  button:not(.print-keep), .modal-root { display: none !important; }

  /* белый фон, чёрный текст */
  body, html { background: #fff !important; color: #000 !important; }
  * { box-shadow: none !important; }

  /* раскрытые блоки */
  .accordion-content { display: block !important; max-height: none !important; }

  /* лого CSS-mark должен оставаться видим */
  .print-logo { display: block; }

  /* page setup */
  @page {
    size: A4 portrait;
    margin: 1.5cm 1.5cm 2cm 1.5cm;
  }

  /* избегаем разрывов внутри карточки */
  .application-card { page-break-inside: avoid; }
  h1, h2, h3 { page-break-after: avoid; }

  /* мелкий шрифт для оптимизации */
  body { font-size: 11pt; line-height: 1.4; }
}
```

### Кнопка «Печать»

В `ApplicationDetailPage` добавить кнопку **«Печать»** (рядом с кнопками transitions). Onclick → `window.print()`.

```tsx
<Button onClick={() => window.print()} variant="ghost">
  <PrinterIcon /> Печать
</Button>
```

Иконка из lucide-react. Кнопка скрывается через `display: none` в `@media print`.

### Шапка для печати

Отдельная hidden-в-обычном-viewport секция:

```tsx
<header className="hidden print:block print-header">
  <img src="/logo-supersymmetria-black.svg" className="print-logo" alt="" />
  <div className="text-xs">Бюро визуальных коммуникаций</div>
  <div className="ml-auto text-xs">{format(new Date(), "dd.MM.yyyy HH:mm")}</div>
</header>
```

(Tailwind утилиты `print:block`, `print:hidden` нужны — они есть в стандартном Tailwind v3.)

### Подпись

Внизу:

```tsx
<footer className="hidden print:flex mt-12 border-t pt-4">
  <div>Исполнитель: ___________________________</div>
  <div className="ml-auto">Дата: ___________________</div>
</footer>
```

### Что важно

- Печатать **черно-белый** логотип (не цветной). Для этого использовать отдельный `logo-mark-black.svg` или CSS `filter: grayscale(1)` на цветной (но это хуже по качеству — лучше отдельный файл).
- Не показывать SSE-индикатор, breadcrumbs, sidebar nav.
- Все детали заявки должны быть видны без скроллинга — то есть accordion'ы автоматически раскрыты.
- Файлы — превью маленькими (50×50 px), под каждым — URL `https://mstechnics.ru/media/...` (чтобы сканировал QR-приложением можно было перейти).

### Тесты

Vitest не подходит для `@media print` (не рендерит print preview). Smoke руками:
- Открыть `/applications/{id}` в Chrome → Cmd+P → сохранить PDF → проверить вид.
- Скриншот PDF в отчёт T-7-013.

## Что НЕ делать

- Не делать отдельную страницу `/applications/:id/print`. Это лишний роут, лишний URL, лишний maintenance.
- Не использовать PDF-генератор на бэкенде (`weasyprint`, `wkhtmltopdf`). Браузерный `window.print()` достаточен для нашего объёма.
- Не вставлять полные изображения файлов — только thumbnails + URL.
- Не печатать **выезды**, **карточки экранов**, **отчёты ЗИП** — отдельно не запрошено, отдельные задачи если потребуется.
