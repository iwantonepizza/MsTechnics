# T-8-108. Prod media reconciliation

> **Статус:** ready
> **Исполнитель:** —
> **Дата создания:** 2026-06-04
> **Приоритет:** P1 post-hotfix
> **Scope:** Runtime media + владелец данных + Django admin

## Контекст

После cutover nginx корректно обслуживает `/media/`, но prod-БД содержит ссылки на файлы, которых
нет ни в активном media-каталоге, ни в сохранённой pre-cutover копии.

Аудит 2026-06-04:

| Модель / поле | Ссылок в БД | Отсутствует файлов |
|---|---:|---:|
| `Display.file` | 8 | 3 |
| `Display.project_photo` | 8 | 7 |
| `Wires.photo` | 17 | 10 |
| `Hubs.photo` | 5 | 1 |
| `Lamels.photo` | 3 | 1 |

## План

1. Выгрузить owner-facing список записей с отсутствующими файлами без публикации приватных URL.
2. Владелец выбирает для каждой записи: повторно загрузить файл или очистить устаревшую ссылку.
3. Выполнить исправление через Django admin/поддерживаемый API.
4. Повторить filesystem-vs-DB audit и проверить отсутствие новых media-404 в nginx.
5. Добавить периодическую проверку broken media references.

## Критерии приёмки

- [ ] Для каждой отсутствующей ссылки принято решение владельца.
- [ ] В БД нет ссылок на отсутствующие media-файлы.
- [ ] `/media/` не отдаёт 404 для файлов, отображаемых в SPA.
- [ ] Проверка broken media references документирована или автоматизирована.

## Что не делать

- Не очищать ссылки и не подменять файлы без решения владельца.
- Не коммитить runtime media в git.
