# T-8-113. Owner feedback: mobile tasks, activity period, upload media type, active application counts

Статус: in-progress
Исполнитель: GPT-5
Дата старта: 2026-06-05

## Контекст

После production стабилизации владелец передал новый пакет UX/API проблем:

1. Блок «Последние действия» на главной должен иметь два периода: `Сегодня` и `Всё время`; по умолчанию `Сегодня`.
2. При загрузке фото/проекта/электросхемы экрана возникает DRF ошибка `Неподдерживаемый тип данных "application/json"`.
3. Счётчик заявок на карточках экранов в списке городов/экранов не должен учитывать архив.
4. На мобильном не нажимаются задания.
5. Для отдела `control` на мобильной главной нужен быстрый блок ежедневных задач ближе к верху страницы.

## Архитектурное решение

- Период активности остаётся frontend-фильтром через существующий API параметр `since`; новый endpoint не нужен.
- Ошибка upload чинится централизованно в `apiClient`: для `FormData` нельзя оставлять default `Content-Type: application/json`.
- Счётчик заявок исправляется на backend через `Count(..., filter=...)`, чтобы UI получал правильное значение независимо от клиента.
- Ежедневные задачи становятся интерактивными для `control` и `monitoring`; это изменение правил доступа фиксируется в backend permission.
- Мобильный блок задач добавляется только для роли `control` на главной и скрывается на desktop.

## Критерии приёмки

- [x] `ActivityFeedBand` по умолчанию грузит события с начала текущего дня.
- [x] В блоке есть переключатели `Сегодня` и `Всё время`; `Всё время` не отправляет `since`.
- [x] Upload `FormData` уходит без `application/json` content-type.
- [x] `application_count` не считает `archive_done` и `archive_unable`.
- [x] `control` может выполнить ежедневную задачу.
- [x] На мобильной главной для `control` есть быстрый блок ежедневных задач над основными колонками.
- [x] Backend/frontend тесты и build пройдены.
- [ ] Изменения запушены и развёрнуты на production через git.
