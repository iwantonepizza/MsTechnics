# T-8-114. Mobile monitoring daily tasks on main page

> **Тип задачи:** frontend
> **Приоритет:** P1
> **Оценка:** 1.5 часа
> **Фаза:** 8
> **Статус:** done
> **Исполнитель:** GPT-5

---

## Цель

Дать мониторингу быстрый доступ к ежедневным задачам с телефона прямо с главной страницы, чтобы смена могла открыть `/menu` и выполнить задания без перехода в экран отдела.

---

## Контекст

- В `frontend/src/widgets/daily-tasks/DailyTasksPanel.tsx` уже есть интерактивный блок ежедневных задач.
- В `frontend/src/pages/display-view/DisplayViewPage.tsx` блок используется в рабочем экране мониторинга/контроля.
- В T-8-113 на главной был добавлен mobile-only блок для `control`; новый owner feedback просит такой быстрый блок именно для пользователей, у которых в профиле есть отдел `monitoring`.
- Multi-role уже есть на backend: `/api/v1/me` возвращает `permission` и `roles`, поэтому проверка должна учитывать оба источника.

---

## План работ

1. Проверить текущий рендер `DailyTasksPanel` на главной и существующие тесты.
2. Переименовать мобильный блок с control-семантики на monitoring-семантику.
3. Показывать блок только если пользователь имеет отдел `monitoring` через `permission` или `roles`.
4. Оставить блок только на мобильных viewport через `md:hidden`, в обычном потоке страницы под KPI, чтобы он не перекрывал другие блоки.
5. Обновить unit-тесты для проверки доступа по `permission` и `roles`.
6. Прогнать frontend-тесты, build и `scripts/check_encoding.py`, чтобы убедиться, что русский текст не повреждён.
7. Создать отчёт в `ai-docs/08-reports/T-8-114.md`, запушить и развернуть на production.

---

## Критерии приёмки

- [x] На мобильной главной `/menu` виден блок ежедневных задач для пользователей с отделом `monitoring`.
- [x] Для `control`, `service` и пользователей без monitoring блок не показывается.
- [x] Пользователь с multi-role, где среди ролей есть `monitoring`, видит блок.
- [x] На desktop блок скрыт CSS-классом `md:hidden`.
- [x] Блок не использует absolute/fixed positioning и не перекрывает KPI, колонки и ленту действий.
- [x] Русский текст в изменённых файлах валидный UTF-8 без mojibake.
- [x] Тесты и build зелёные.
- [x] Production deploy выполнен.

---

## Что НЕ нужно делать

- Не менять API ежедневных задач.
- Не менять FSM заявок и backend-права.
- Не добавлять новый дизайн блока вместо существующего `DailyTasksPanel`.
- Не трогать desktop-компоновку главной страницы.
