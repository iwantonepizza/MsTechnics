# Security Conventions

Этот файл фиксирует базовые правила гигиены для репозитория MsTechnics. Если артефакт нельзя безопасно показать постороннему человеку, ему не место в git.

---

## Что не коммитим

- Дампы БД: `*.dump`, `*.sql`, `*.sql.gz`, `db_dumps/`, `dumps/`
- Логи локальных прогонов и миграций: `logs/`, `*.log`
- Локальные секреты и ключи: `.env`, `Config/.env`, `client_secret.json`, `Config/client_secret.json`, `token.pickle`, `*.pickle`
- Build artefacts: `*.egg-info/`, `staticfiles/`, `htmlcov/`, `.coverage*`

Если нужен пример конфигурации, держим только обезличенные шаблоны вроде `.env.example`.

---

## Что делать при утечке

1. Сразу убрать файл из индекса через `git rm --cached`.
2. Добавить путь в `.gitignore`, чтобы утечка не повторилась.
3. Проверить историю `git log --all --full-history -- <path>`.
4. Если файл попал в remote, согласовать `git filter-repo` и force-push с владельцем.
5. Если утекли реальные секреты, сначала ротировать их, потом чистить историю.

`git rm` в новом коммите не лечит историю. Если секрет уже был pushed, он считается скомпрометированным.

---

## Правила для прод-данных

- Прод-дампы и выгрузки с PII разрешены только как локальные временные артефакты.
- Использовать их можно для миграционных прогонов и smoke-тестов, но не хранить в репозитории.
- Любые отчеты по таким прогонам должны быть обезличены: без JWT, паролей, токенов и тел ответов с персональными данными.

---

## Проверка перед PR

- `git status` не показывает дампы, логи, локальные секреты и build artefacts.
- `git ls-tree -r HEAD -- <path>` не показывает прод-артефакты в текущем `HEAD`.
- Для инцидентов с историей git в отчете явно указано, нужен ли `filter-repo`.
