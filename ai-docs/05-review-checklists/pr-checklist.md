# PR Checklist (краткий, для автора)

Перед тем как нажать «Open PR», убедись что:

## Минимум

- [ ] Ветка от свежей `main` (rebase сделан)
- [ ] Все коммиты соответствуют `commit-format.md`
- [ ] В PR-описании есть: **что**, **зачем (ссылка на задачу)**, **как**, **чеклист**
- [ ] Ссылка на отчёт в `ai-docs/08-reports/<task-id>.md`

## Локальные проверки

- [ ] `pre-commit run --all-files` — зелёное
- [ ] `pytest` — зелёное
- [ ] `mypy` — зелёное
- [ ] Ручное тестирование функции — сделано
- [ ] Нет debug-кода / console.log / print / закомментированных кусков

## Содержание

- [ ] Ветка решает **одну** задачу, не две сразу
- [ ] Сколько файлов изменено — понятно зачем каждый
- [ ] Нет случайных изменений (.idea/, .vscode/, локальные конфиги)

## Если есть миграции

- [ ] Миграция протестирована на копии прод-БД
- [ ] В описании PR указан размер затронутых таблиц
- [ ] Описан план отката

## Если есть новый endpoint

- [ ] Обновлён `ai-docs/07-frontend/api-contract.md`
- [ ] Схема (drf-spectacular) обновилась, Swagger показывает
- [ ] Фронту сообщено (если он уже в параллели)

## Если изменил архитектуру

- [ ] Обновлён соответствующий файл в `ai-docs/00-architecture/`
- [ ] Если значимое решение — создан ADR в `ai-docs/adr/`

## Если хотфикс прода

- [ ] После мёржа — проверь что деплой прошёл
- [ ] Post-mortem в `ai-docs/08-reports/incident-YYYY-MM-DD.md`

## Что пишу в отчёт

- [ ] Что сделано — по шагам из задачи
- [ ] Что не получилось / отклонения
- [ ] Тесты: файлов, тестов, coverage
- [ ] Оценка vs факт по времени
- [ ] Дальнейшие шаги / рекомендации

---

## Что ревьюер спросит

- «Зачем это поле в таблице?» — отвечай бизнес-логикой, не «так было нужно»
- «Почему не вынес в сервис?» — отвечай заранее в описании, если сделал иначе
- «Тест на граничный случай X?» — заранее добавь тест
- «Это точно обратно совместимо?» — проверь перед PR

---

## Маркеры качественного PR

- PR-описание длиннее diff'а
- Тесты читаются лучше кода
- Ты сам прошёл по чеклисту до открытия PR
- Нет «fix style» / «fix tests» / «fix review comments» коммитов — они сквашены
