# Суперсимметрия — brand guidelines для внутреннего UI

Полный свод правил применения бренда «Суперсимметрия» к нашему **внутреннему рабочему инструменту** (система управления LED-панелями). Основано на:

- `Брендбук. Суперсимметрия. 2026.pdf` v1.0 (115 страниц, Be Art Agency)
- `Гайдбук_Суперсимметрия_2026_Ver_2_0.pdf` v2.0 (50 страниц, источник практических правил)

Оба PDF лежат в `dumps/brand-pdf/` (не коммитим — это вендорские артефакты на ~100 МБ). Извлечённый текст — `brendbook-fitz.txt` и `guidebook-fitz.txt`.

Этот файл — **выжимка** для разработчиков, не пересказ PDF целиком. PDF при имплементации T-7-001/T-7-002 надо открыть **рядом** с этим документом.

---

## 1. Что такое Суперсимметрия

**Суперсимметрия** — это DOOH-агентство (Digital Out-of-Home), которое управляет рекламными медиафасадами и панелями в городской среде. Старое имя компании — **msgroup**, отсюда `mstechnics` в коде и БД. Наш внутренний инструмент — это **система обслуживания** их собственных рекламных панелей: мониторинг состояния, заявки сервисменам, ЗИП-склад.

| | |
|---|---|
| **Имя** | Суперсимметрия |
| **Слоган** | «Соединяем важное» |
| **Дескриптор** | «Бюро визуальных коммуникаций» |
| **Манифест** | «С нами город оживает по-настоящему» |
| **Большая идея** | «Искусство созидать» |
| **Голос** | Вдохновляющий / Креативный / Уверенный / Открытый / Многослойный. **Сдержанно**, без перегруза. |

### Где это применяется в нашем UI

| Артефакт | Что используем |
|---|---|
| `<title>` | «Суперсимметрия — система управления экранами» |
| `<meta name="description">` | «Бюро визуальных коммуникаций — внутренняя система» |
| Header logo | Знак + текст «Суперсимметрия» |
| Login page | Большой логотип + subtitle «Соединяем важное» |
| Footer | «Суперсимметрия · Бюро визуальных коммуникаций · {year}» |
| Favicon, PWA manifest | Знак (favicon только знак, не текст) |
| Email templates | Подпись: «Суперсимметрия» |

**Сдержанно:** Internal tool — это рабочая среда. Не превращаем UI в маркетинговую витрину. Стилеобразующие элементы из брендбука (рамка «машинного зрения», маркерные выделения, плашки с пиксельным шрифтом) — **для маркетинговых материалов и сайта-витрины, не для табличных интерфейсов**.

## 2. Цвета

Точная палитра + CSS-mapping — в [`brand-palette-supersymmetria.md`](brand-palette-supersymmetria.md) (уже существует). Здесь — правила применения.

**Основные** (большие площади, фоны, текст):
- Призрачно белый `#fafcff`, Платина `#e1e5ea`, Серебро `#b8bfc6` — light theme.
- Ночное небо `#040f1d`, Ночная дымка `#3c4856` — dark theme.

**Акцентные** (точечно, для CTA-кнопок и активных состояний):
- Солнечный луч `#ffea00` — primary action.
- Ночное окно `#fdd734` — hover на primary.
- Закат `#fdb734` — warning / attention.

### Правила применения

**Можно:**
- Акцентный цвет на primary-кнопках, активных tab'ах, индикаторах SSE-подключения.
- Жёлтый highlight на тексте поиска (`<mark>`) — это «маркерное выделение» из брендбука.
- Жёлтые тени-обводки фокуса (focus ring) на полях ввода.

**Нельзя:**
- Большие заливки акцентным жёлтым (Cards, Modals, Side panels). Палитра явно говорит: «Не используйте их в качестве заливок».
- Логотип на жёлтом фоне.
- Перекрашивать логотип в акцентные цвета.

### Не трогаем (как было)

- **Цвета состояний панелей** (`Condition.color` в БД) — это данные владельца, не дизайн-токены.
- **Семантические alerts** — красный для `danger`, жёлтый для `warning`, зелёный для `success`, синий для `info`. Они существуют независимо от brand palette.

## 3. Шрифты

Брендбук определяет 3 шрифта. Для нашего UI применимы 2 из 3.

| Шрифт | Назначение в брендбуке | Применение в UI |
|---|---|---|
| **TT Travels Text Bold** | акцидентный №1, заголовки, брендлайны | ✅ H1/H2/H3 заголовки страниц + кнопок CTA |
| **Biform Pixel** | акцидентный №2, пиксельный, **только в плашках** | ❌ **НЕ использовать** — для маркетинговых баннеров и сайта-витрины, не для табличных UI |
| **Biform Regular** | наборный, основной текст | ✅ body, label, paragraph, таблицы |

### Установка

✅ **TT Travels** (2026-05-17) получен от владельца как `TTTravels.zip` — лежит в `frontend/public/fonts/tt-travels/` (36 файлов: 11 весов × WOFF + TTF, итого ~3 MB). Архив `dumps/tt-travels-unzip/` — в `.gitignore`, не коммитим.

> **Расхождение с гайдбуком:** в гайдбуке указано «**TT Travels Text** Bold» (typetype.ru), а у владельца файлы «**TT Travels**» (без «Text», family-name `'TT Travels'`, сгенерировано Transfonter в 2017). Это **родственные шрифты, не идентичные**. TT Travels Text оптимизирован для длинных текстов (более узкие буквы, увеличенный x-height); TT Travels — для дисплейного использования. На практике для нашего UI разница не критична. Если в будущем владелец предоставит «TT Travels Text» — заменим. Сейчас используем что есть.

🟡 **Biform Regular** (paratype.ru) — лицензия есть, но **файлы не получены**. Запросить у владельца отдельно. До получения — fallback на `'TT Travels'` (400 weight) + `system-ui`.

🟡 **WOFF2** — в архиве владельца только WOFF + TTF. WOFF2 сжимает на 30% лучше. Кодер T-7-002 должен сконвертировать WOFF → WOFF2 (через `woff2_compress` или `fonttools`), иначе FCP/LCP пострадают.

### `@font-face` подключение

В `frontend/src/app/styles/fonts.css` (создать в T-7-002):

```css
/* TT Travels — для заголовков и accent элементов */
@font-face {
  font-family: "TT Travels";
  src: url("/fonts/tt-travels/TTTravels-Regular.woff2") format("woff2"),
       url("/fonts/tt-travels/TTTravels-Regular.woff") format("woff"),
       url("/fonts/tt-travels/TTTravels-Regular.ttf") format("truetype");
  font-weight: 400;
  font-display: swap;
}

@font-face {
  font-family: "TT Travels";
  src: url("/fonts/tt-travels/TTTravels-Medium.woff2") format("woff2"),
       url("/fonts/tt-travels/TTTravels-Medium.woff") format("woff"),
       url("/fonts/tt-travels/TTTravels-Medium.ttf") format("truetype");
  font-weight: 500;
  font-display: swap;
}

@font-face {
  font-family: "TT Travels";
  src: url("/fonts/tt-travels/TTTravels-Bold.woff2") format("woff2"),
       url("/fonts/tt-travels/TTTravels-Bold.woff") format("woff"),
       url("/fonts/tt-travels/TTTravels-Bold.ttf") format("truetype");
  font-weight: 700;
  font-display: swap;
}

/* остальные веса (Thin/Light/DemiBold/ExtraBold/Black + italic-вариации)
   подключать по мере необходимости — пока не используем */
```

Reference: `dumps/tt-travels-unzip/stylesheet.css` — оригинальный CSS от Transfonter с полным списком весов.

### CSS variables

```css
:root {
  --font-display: "TT Travels", system-ui, "Segoe UI", sans-serif;
  /* Body временно использует TT Travels Regular до получения Biform */
  --font-body: "TT Travels", system-ui, "Segoe UI", sans-serif;
  --font-mono: "JetBrains Mono", "Cascadia Mono", Consolas, monospace;
}
```

`system-ui` — safety net на случай, если файл шрифта не загрузился. Inter в fallback **не используем**: у нас есть лицензированные брендовые шрифты.

После получения Biform — заменить `--font-body`:

```css
--font-body: "Biform", "TT Travels", system-ui, "Segoe UI", sans-serif;
```

### Логотип — получен

✅ **SVG получен** (2026-05-17) — лежит в `frontend/public/logo-supersymmetria.svg`. Особенности:

- `viewBox="0 0 600 300"` — горизонтальная версия (знак + надпись «супер симметрия»).
- `fill="currentColor"` — цвет управляется CSS-свойством `color` родителя. Отлично работает для light/dark тем без двух разных файлов.
- Default color: `#030304` (почти Ночное небо `#040f1d`).
- `role="img"` + `aria-labelledby` + `<title>` — доступность OK.

В Header:

```tsx
<img src="/logo-supersymmetria.svg" alt="Суперсимметрия" className="h-8 text-fg-0 dark:text-fg-0" />
```

Так как SVG inline-загружается через `<img>`, `currentColor` не работает — нужно либо встроить SVG как React-компонент, либо использовать CSS `filter: invert(1)` для dark theme. **Правильнее — inline SVG**, тогда `color: var(--fg-0)` работает напрямую:

```tsx
import LogoSvg from "/logo-supersymmetria.svg?react"   // через vite-plugin-svgr или подобный
...
<LogoSvg className="h-8 text-fg-0" />
```

Если `vite-plugin-svgr` не подключён — в T-7-002 подключить его (`npm i -D vite-plugin-svgr`) и обновить `vite.config.ts`.

### Что ещё нужно от дизайнера (не блокирует T-7-001)

- **Только знак** (без надписи) для favicon — вырезать из текущего SVG (первый `<path>` отвечает за знак, остальные за буквы).
- **Чёрно-белые fallback-варианты** — на случай печати или окружения без CSS.
- **PWA-иконки** 192/512 PNG — генерируются из SVG.

### Иерархия

```css
h1 { font-family: var(--font-display); font-size: 32px; font-weight: 700; }
h2 { font-family: var(--font-display); font-size: 24px; font-weight: 700; }
h3 { font-family: var(--font-display); font-size: 18px; font-weight: 700; }
body { font-family: var(--font-body); font-size: 14px; }
.label { font-family: var(--font-body); font-size: 12px; }
.mono { font-family: var(--font-mono); }   /* IDs панелей, timestamps */
```

## 4. Логотип

### Версии

Брендбук — это «полиморфный логотип»: знак (два пиксельных модуля) + текстовая часть «супер / симметрия» в две строки. Есть **горизонтальная** и **вертикальная** версии.

Для нашего UI:
- **Header** — горизонтальная версия (знак + надпись справа), высота 32-40 px.
- **Login page** — горизонтальная или вертикальная, ширина ~200-300 px.
- **Favicon** — **только знак** без текста (минимум 16×16 px).
- **PWA icon** (192/512) — знак + надпись.

### Цвета логотипа

**Только чёрный или белый.** В light theme — чёрный (`#040f1d` — Ночное небо). В dark theme — белый (`#fafcff` — Призрачно белый).

### Минимальные размеры

- Знак отдельно: **16 px** ширина (для favicon).
- Логотип с надписью: **100 px** ширина.
- С дескриптором («Бюро визуальных коммуникаций»): **150 px** ширина.

### Охранное поле

Свободная зона вокруг логотипа = половина высоты знака. В Header и Login это значит: не лепить текст/кнопки/другие элементы вплотную.

### Что нельзя

Гайдбук стр. 15:
- Растягивать по горизонтали/вертикали (использовать только пропорционально).
- Наносить градиенты, тени, обводку.
- Поворачивать.
- Размещать на акцентной заливке (жёлтый фон).
- Размещать на низкоконтрастном фоне.
- Перекрашивать в акцентные цвета.

### Где взять файл

Владелец прислал PNG в чате. Дизайнер векторизует в SVG.
Положить:
- `frontend/public/logo-supersymmetria.svg` (полная горизонтальная версия)
- `frontend/public/logo-mark.svg` (только знак)
- `frontend/public/favicon.ico` + `.png` варианты
- `frontend/public/apple-touch-icon.png`

## 5. Стилеобразующие элементы

Брендбук определяет:

| Элемент | Назначение | Применять в нашем UI? |
|---|---|---|
| Маркерное выделение | жёлтый highlight на ключевых словах | ✅ search match (`<mark>`), notifications counter |
| Цветная подложка с Biform Pixel | плашка-акцент в плакатной графике | ❌ только если будет PDF-отчёт со «спецификацией панелей» — там можно |
| Фирменная рамка («машинное зрение») | акцент на изображениях/картах | ⚠️ опционально на «карточке экрана» — рамка вокруг превью | 
| Пиктограммы пиксельные | иконки в брендовых материалах | ⚠️ опционально — но lucide-react (текущие иконки) проще и универсальнее |
| Градиенты (Солнечный луч, Закат, Свет ночного окна) | инфографика | ⚠️ только в Dashboard / charts (T-6-003 observability) |

**Правило:** в рабочем интерфейсе (DisplayView, ApplicationList, ZipPage) — **никаких** стилеобразующих элементов поверх данных. Они для маркетинговых страниц компании, а не для управления панелями.

## 6. Голос бренда в UI-копирайте

Брендбук стр. 65-66: голос **Вдохновляющий, но не навязчивый. Уверенный, но не высокомерный. Открытый, но не покорный.**

В нашем UI это значит:

| Хорошо | Плохо |
|---|---|
| «Заявка создана» | «Создание заявки прошло успешно!» (избыточно) |
| «Соединяем важное» (login subtitle) | «Лучшее решение для управления экранами!» (хвастовство) |
| «Точно?» (confirm prompt) | «Вы уверены, что хотите выполнить это критически важное действие?» (перегруз) |
| «Не удалось войти. Проверьте логин и пароль» | «ОШИБКА АВТОРИЗАЦИИ: некорректные учётные данные» (высокомерно) |

Локализация — **только русский**, на «вы», без сленга.

## 9. Что НЕ берём из брендбука в UI

Брендбук содержит много для **компании** как для бренда: позиционирование, ценностное предложение, стейкхолдеры, манифест, легенда (msgroup → Суперсимметрия), мерч, визитки, конверты, наружные таблички. Это **не для нашего инструмента**.

Если в будущем будет публичный сайт компании / клиентский кабинет (для рекламодателей) — там брендбук применяется целиком. Внутренний инструмент — только палитра + шрифты + логотип + название.
