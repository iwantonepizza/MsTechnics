## Раунд 2 (2026-05-17) — закрытие open questions

| # | Вопрос | Ответ владельца | Архитектурная интерпретация |
|---|---|---|---|
| 1 | Лицензии на шрифты TT Travels Text + Biform? | **Есть** | T-7-002 подключаем шрифты напрямую (без Inter fallback в качестве основного). Inter оставляем только как `font-family` system fallback на случай проблемы загрузки. |
| 2 | SVG-логотип | **«сделаю позже»** | T-7-001 остаётся **blocked** до получения SVG. Можно начать с PNG-плейсхолдером из чата, но не финализировать. |
| 3 | VK Workspace API — что это, для чего? | **«что это и для чего вообще, не пойму»** | Владелец **не использует** VK Workspace (это корпоративный мессенджер VK для бизнеса). Значит, в раунде 1 ответ A7 «MAX → VK Workspace → VK → TG» был неточным. Решение архитектора: **убираем VK Workspace** из плана. Финальный fallback chain: **MAX → Telegram → Email**. Канал «VK Community Bot» добавим, когда владелец заведёт community-бота (T-7-021 остаётся blocked). |
| 4 | Default-state при «снятии панели» (T-7-007) | **«если снимется панель просто, то у неё то состояние, что было, её же не отметили как сломанную»** | Различаем **2 сценария**: (а) снятие в рамках заявки — `Condition` обязательно к указанию сервисменом (это закрывает DV-S6); (б) ручное снятие без заявки — `Condition` остаётся прежним, в форме поле необязательное. См. T-7-007. |
| 5 | Пороги `low_stock_threshold` для расходников | **3** (default для всех) | T-7-005 (Storage модели) добавляет поле `low_stock_threshold = IntegerField(default=3)` на `Wires/Hubs/Lamels/PowerBlocks/Connectors`. UI подсвечивает красным при `count < low_stock_threshold`. Админ может менять threshold на каждой записи. |
| 6 | VK bot token — есть? | **Нет**, аккаунт **личный** | T-7-021 остаётся blocked. Архитектор рекомендует **завести community group** (через `vk.com/groups → создать → подключить bot API`) — это правильная практика; личный аккаунт можно случайно забанить, и его API ограничен. До этого канал VK не реализуем. |

### Корректировки в зависимостях

- **T-7-020 (VK Workspace channel)** — **CANCELLED** (не нужен по факту).
- **T-7-021 (VK обычный channel)** — остаётся **blocked** до community-бота.
- **T-7-022 (channel fallback reorder)** — упрощается: новый chain `MAX → Telegram → Email`. Карточка обновится после T-7-021 done.
- **T-7-001 (rebranding)** — **blocked** до SVG-логотипа (frontend может начать с PNG для smoke).
- **T-7-005 (Storage)** — **ready**, default threshold = 3 закрыт.
- **T-7-007 (поведение при снятии)** — **ready**, два сценария чётко определены.
