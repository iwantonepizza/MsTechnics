## Что сделано

- Подтверждено, что prod JSON logging уже включается через `structlog.processors.JSONRenderer()` при `DEBUG=False`.
- Добавлен runtime dependency `sentry-sdk[django]>=2.59,<3`.
- Добавлен optional Sentry init в `Config/settings.py`: SDK стартует только если задан `SENTRY_DSN`.
- `send_default_pii=False`; в Sentry явно передаётся только `user_id`.
- Добавлен `shared.middleware.RequestContextMiddleware`:
  - генерирует/читает `X-Request-ID`;
  - добавляет `request_id` и `user_id` в structlog contextvars;
  - добавляет `X-Request-ID` в response;
  - прокидывает `request_id`/`user_id` в Sentry scope.
- Добавлены env placeholders в `.env.example`.
- Добавлена docker `json-file` log rotation для `web`: `10m` x `5`.

---

## Проверки

- `python -m compileall -q Config/settings.py shared/middleware.py` — зелено.
- `$env:DJANGO_SETTINGS_MODULE='Config.settings'; $env:SENTRY_DSN=''; .venv\Scripts\python -c "import django; django.setup(); ..."` — зелено, middleware подключён.
- `rg` подтверждает наличие `SENTRY_DSN`, `RequestContextMiddleware`, docker logging и `sentry-sdk` в dependency files.

Что не проверено:
- Реальная отправка события в Sentry — нужен настоящий `SENTRY_DSN`.
- Backend pytest — в текущей `.venv` нет `pytest`.
