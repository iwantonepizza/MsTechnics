## Архитектурные решения

**T-2-028:** `Panel.application_status` как property даёт правильный UX без field,
но создаёт N+1 при итерации по панелям. Решение — `with_application_status()` annotation в views.
Все view в `apps/` должны использовать аннотацию, не property.

**T-2-025 atomic=False:** Backfill миграции помечены `atomic=False` — позволяют прерываться
и перезапускаться без корраптинга данных (идемпотентный WHERE IS NULL).

**T-2-030:** `status_legacy` поле оставлено временно для отката. Удалить в T-2-030-followup
через 2 недели после успешного деплоя.
