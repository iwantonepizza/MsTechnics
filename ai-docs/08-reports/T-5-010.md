## Архитектурные решения

- Health-check сделан отдельным script в `scripts/`, потому это инфраструктурная проверка, запускаемая из CI/cron без Django boot.
- Вывод health-check — JSON, чтобы его можно было читать мониторингом.
- Proxy URL в выводе маскируется, чтобы случайно не утек пароль.
