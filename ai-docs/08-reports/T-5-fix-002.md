## Что сделано

- Восстановлен рабочий `pip install -e ".[dev,test]"`: `pyproject.toml` дополнен настройкой package discovery для setuptools.
- `requirements.txt` переписан в UTF-8 без BOM по содержимому `requirements.lock`.
- `Makefile` и `scripts/bootstrap_dev.sh` теперь ставят `.[dev,test]`, чтобы новое окружение поднималось одинаково.
- Для актуального `pytest-cov` конфиг переведён на `.coveragerc`; из `pyproject.toml` убран сломанный `--cov-omit`.
- `pytest --collect-only` теперь реально собирает тесты, а не падает на старом конфиге.
- В `ai-docs/04-conventions/code-style.md` добавлена короткая инструкция по dev extras.

## Отклонения от плана

- Пришлось править не только bootstrap, но и `pyproject.toml`: без package discovery editable install был сломан.
- `ruff`, `black` и `mypy` запускаются, но показывают большой накопленный baseline; это вынесено в отдельный follow-up, а не исправлялось в этом hotfix.
- `pre-commit run --all-files` невалиден в текущем workspace: каталог не является git-репозиторием.

Нет других отклонений.

---

## Архитектурные решения

- `requirements.txt` синхронизирован именно с `requirements.lock`, а не с extras: runtime и dev/test зависимости разделены.
- Покрытие pytest-cov вынесено в `.coveragerc`, потому что CLI-опция `--cov-omit` больше не поддерживается в используемой версии плагина.
- Линт-бейзлайн не исправлялся "заодно", чтобы не смешивать инфраструктурный hotfix с массовым форматным churn.
