# T-5-fix-003 follow-up — schema alignment tail on restored prod dump

## Что всплыло

При повторной manual acceptance попытке для `T-7-100` на restored prod-дампе backend падал не на фронтовой логике, а на старом schema drift:

- `application.status_id` физически оставался `text`, хотя Django state уже ожидал FK на `application_status(id)`.
- После фикса этого слоя вскрылись ещё не доведённые legacy string-FK:
  - `application_status.{color_id,color_text_id,icon_id}`
  - `condition.{color_id,color_text_id,icon_id}`
  - `department.{color_id,color_text_id,icon_id}`

Именно из-за этого `DashboardView` ронял restored prod-copy с `ProgrammingError: operator does not exist: character varying = bigint`.

## Что сделано

- Добавлена forward-only миграция [apps/workflow/applications/migrations/0006_convert_application_status_fk_to_id.py](/abs/path/C:/Users/miser/mstechnics/mstechnics/apps/workflow/applications/migrations/0006_convert_application_status_fk_to_id.py):
  - конвертирует `application.status_id` из status-name в `application_status.id`
  - конвертирует `application_status.color_id/color_text_id` по `color.name`
  - конвертирует `application_status.icon_id` по `smile.smile_icon`
- Добавлена forward-only миграция [apps/core/references/migrations/0003_convert_condition_department_style_fks_to_id.py](/abs/path/C:/Users/miser/mstechnics/mstechnics/apps/core/references/migrations/0003_convert_condition_department_style_fks_to_id.py):
  - конвертирует `condition` и `department` style-FK на `color(id)` и `smile(id)`
- Добавлен regression test [apps/interface/tests/test_schema_alignment.py](/abs/path/C:/Users/miser/mstechnics/mstechnics/apps/interface/tests/test_schema_alignment.py), который через Django introspection проверяет, что критичные physical FK-колонки больше не строковые.

## Проверка

- `ruff check apps/workflow/applications/migrations/0006_convert_application_status_fk_to_id.py apps/core/references/migrations/0003_convert_condition_department_style_fks_to_id.py apps/interface/tests/test_schema_alignment.py`
- `pytest apps/interface/tests/test_schema_alignment.py`
- На временном PostgreSQL-контейнере с restore `db_dumps/mstechnics.dump`:
  - до `migrate`: `application.status_id`, `application_status.*`, `condition.*`, `department.*` были `text/varchar`
  - после `migrate`: все перечисленные колонки стали `bigint`
  - `DashboardView` через `APIRequestFactory` → `200`
  - `DisplayViewSet.list` → `200`
  - `PanelViewSet.list` → `200`

## Вывод

Первоначальный `T-5-fix-003` действительно оставил недоконвертированный хвост legacy string-FK. Этот follow-up закрывает именно тот physical drift, который мешал честно принимать `T-7-100` на restored prod-copy.
