- **Почему:** в текущей SPA такого маршрута и страницы нет; фактическая карточка заявки живёт внутри `DisplayViewPage` как detail-pane выбранной заявки. Реализация привязана к реальной текущей точке входа, чтобы не вводить новый маршрут без архитектурной постановки.

## Архитектурные решения

- Printable-версия сделана через `@media print` поверх существующей карточки, без отдельного `/print` URL и без backend PDF-генерации.
- Для снятия панели из карточки заявки используется фактическая панель из `display.cells`, потому что `ApplicationDetail.panel` в API содержит только `PanelMini` и не подходит под `PanelLike` для `PanelRemovalModal`.
