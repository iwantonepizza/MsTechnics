## T-7-011 — Колокольчик уведомлений (X2)

**Backend (новое):**
- `apps/interface/api/v1/notifications/` — новый модуль (`__init__.py`, `urls.py`, `views.py`, `serializers.py`).
- `GET /api/v1/notifications/inbox/?limit=20` — возвращает последние N уведомлений текущего юзера со статусом `sent`/`pending` по `-created_at`. `select_related(related_target_ct)`.
- Подключение: `apps/interface/api/v1/urls.py` → `path("", include("apps.interface.api.v1.notifications.urls"))`.

**Архитектурное решение: read-state на клиенте.**
Модель `Notification` не имеет поля `read_at`. Чтобы не делать BC-breaking миграцию, `last_seen_id` хранится в **localStorage** клиента. Юзер открывает popover → `markAllSeen` → `localStorage['notifications.lastSeenId'] = maxId`. При следующем фетче если есть id > lastSeenId — бейдж снова поднимается. Минус: read-state per-device, не sync через сервер. Плюс: ноль миграций, инкрементальный rollout.

**Frontend:**
- `frontend/src/entities/notification/hooks.ts` — `useInbox(limit)` через React Query, refetchInterval 30s, staleTime 10s, экспорт `markAllSeen()`.
- `frontend/src/widgets/navigation/NotificationBell.tsx` — компонент колокольчика: иконка Bell/BellRing (зависит от unreadCount), red badge с `9+` для >9, popover (250px wide) с `<NotificationRow>`, deep-links на application/departure/panel target. Закрытие по клику вне.
- Подключён в Header рядом с SSEDot.

**Tests:** `NotificationBell.test.tsx` — 6 (рендер без badge, badge с числом, открытие → markAllSeen → localStorage, empty-state, deep-link на application, lastSeenId persist).

## T-7-033 — DnD ZIP колонок (Z2)

**Архитектурное решение: HTML5 native DnD, без новых зависимостей.**
@dnd-kit и подобные тащат ~30 KB minified. Для четырёх колонок и простого move-сценария нативный API избыточен достаточен. Custom MIME `application/x-panel-id` гарантирует что drop в чужие элементы (текст с другой страницы) не сработает.

**Frontend:**
- `frontend/src/pages/zip/ZipPage.tsx`:
  - `PanelChip` теперь `draggable={true}` с `onDragStart` (setData MIME + name + effectAllowed=move).
  - `PanelColumn` — drop target **только** для `zip/hand/service` (`DND_ALLOWED_TARGETS`). `monitor` исключён, потому что установка в ячейку — отдельный flow `replace_panel_in_cell`, а не смена department. `onDragOver` фильтрует по MIME, `onDrop` дёргает `useChangeDepartment.mutateAsync`. Подсветка через `outline: 2px dashed var(--accent)` при `dragHover`.
  - Если панель уже в этом отделе — drop игнорируется.

**Tests:** `ZipPage.test.tsx` — 3 (low-stock хайлайт + 2 новых для DnD): chip draggable, drop в service → mutateAsync({id, department:'service'}); drop на monitor не дёргает mutation.
