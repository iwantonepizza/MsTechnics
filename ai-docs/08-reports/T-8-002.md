# T-8-002 — Матрица роль×кнопка на экране отдела

## Что исправлено

- В `frontend/src/pages/display-view/DisplayViewPage.tsx` собрана единая матрица действий для панели, пустой ячейки и карточки заявки:
  - `monitoring` видит только смену состояния панели, создание заявки на не-`work` панели и удаление своей свежей заявки;
  - `control` не видит create/remove/change-condition/change-department;
  - `service` видит установку/снятие панели и смену отдела/состояния, но не создание заявки;
  - `admin/all` сохраняет полный набор действий.
- Убрана дублирующая кнопка «Создать заявку» из title-bar.
- `AlarmRail` переведён с общего `canCreate` на проверку по конкретной ячейке, чтобы сервис и контроль не видели лишние create-action.
- `ChangeConditionModal` получил `allowedConditionNames`, чтобы экран мог ограничивать набор доступных состояний для мониторинга.

## Причина бага

- До правки экран собирал доступность кнопок из нескольких несвязанных условий.
- Из-за этого monitoring/control видели лишние действия на панели, а create-action не был привязан к фактическому состоянию панели.

## Проверка

- `frontend: npm test -- src/pages/display-view/DisplayViewPage.test.tsx`
- `frontend: npm run typecheck`

## Регрессии

- В `frontend/src/pages/display-view/DisplayViewPage.test.tsx` добавлены проверки:
  - monitoring на `work` панели не видит create-action;
  - monitoring на проблемной панели видит create-action;
  - control не видит мутаций панели;
  - service может ставить панель в пустую ячейку.
