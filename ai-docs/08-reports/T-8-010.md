# T-8-010 — Цветной индикатор состояния экрана

## Что исправлено

- В `apps/interface/api/v1/displays/serializers.py` вычисление `aggregated_condition` переведено с сравнения по `condition.id` на явную severity-модель.
- В `apps/directory/displays/models.py` такой же severity-порядок добавлен в `Display.current_condition`, чтобы логика не расходилась между prefetched и обычным путём.
- В `frontend/src/shared/api/types.ts` зафиксирован `aggregated_condition` в типе `DisplayListItem`.
- В `frontend/src/pages/department/DepartmentListPage.tsx` у карточки экрана добавлен цветной кружок состояния с `title`/`aria-label`.

## Причина бага

- Раньше backend выбирал «худшее» состояние по максимальному `condition.id`.
- Это ломалось при любом порядке заполнения справочника: более тяжёлое состояние могло иметь меньший `id`, и список экранов показывал неверный цвет.

## Регрессии

- В `apps/interface/tests/test_phase7_followups.py` добавлен кейс, где `unrecoverable` создаётся раньше `work`, чтобы проверить выбор по тяжести, а не по `id`.
- В `frontend/src/pages/department/DepartmentListPage.test.tsx` добавлена проверка `aria-label` для цветного индикатора и fallback для экрана без состояния.
