# T-8-031 — Создать заявку только если `condition != work`

## Что исправлено

- Ограничение реализовано в матрице `DisplayViewPage`: для `monitoring` кнопка «Создать заявку» показывается только если состояние панели входит в проблемный набор (`problem/error/broken/unrecoverable`).
- На `work` панели monitoring видит только кнопку смены состояния, без create-action.
- Для `admin/all` создание заявки оставлено доступным как административное действие.

## Причина бага

- Ранее create-action гейтился только по роли и наличию панели, поэтому monitoring мог открыть создание заявки даже на исправной (`work`) панели.

## Проверка

- `frontend: npm test -- src/pages/display-view/DisplayViewPage.test.tsx`
- `frontend: npm run typecheck`

## Регрессии

- В `frontend/src/pages/display-view/DisplayViewPage.test.tsx` добавлены отдельные кейсы для `work` и `error` панели.
