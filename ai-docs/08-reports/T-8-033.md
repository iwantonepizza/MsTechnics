# T-8-033 — Вкладки «Созданные / Все / История» в monitoring

## Что исправлено

- В `frontend/src/widgets/applications-panel/ApplicationsPanel.tsx` monitoring-вкладки переименованы в `Созданные / Все / История`.
- Для вкладки `История` добавлена отдельная лента событий по текущему экрану через `useActivityLog({ display: slug, kind: 'application.' })`.
- Очередь заявок и history-path теперь разведены: при активной вкладке `История` панель не пытается читать `/applications/` как псевдо-box.
- В `frontend/src/entities/application/hooks.ts` `useApplications()` получил флаг `enabled`, чтобы history-tab не дёргал лишний запрос.

## Причина бага

- Monitoring использовал старые названия вкладок и не имел отдельной обзорной истории по всем заявкам экрана.
- Без разведения источников данных история неизбежно превращалась бы в ту же выдачу `/applications/`, только под другим названием.

## Регрессии

- В `frontend/src/widgets/applications-panel/ApplicationsPanel.test.tsx` добавлены проверки:
  - monitoring видит 3 вкладки и получает timeline на `История`;
  - control не получает вкладку `История`.
