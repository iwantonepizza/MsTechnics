# T-8-034 — Видеоблок iframe-камеры в monitoring

## Что исправлено

- В `apps/interface/api/v1/displays/serializers.py` detail API экрана теперь явно отдаёт `camera_link`.
- В `frontend/src/shared/api/types.ts` тип `DisplayDetail` синхронизирован с новым полем `camera_link`.
- В `frontend/src/pages/display-view/DisplayViewPage.tsx` для monitoring добавлен блок камеры под сеткой экрана:
  - iframe открывается прямо в карточке;
  - карточка сворачивается и разворачивается;
  - если встраивание не загрузилось за 5 секунд, показывается фолбэк-кнопка `Открыть камеру`.

## Причина бага

- В данных экрана ссылка на камеру уже существовала на модели `Display`, но detail API её не возвращал, поэтому SPA не могла построить видеоблок.
- Для vendor-страниц с `X-Frame-Options` или CSP без фолбэка iframe создавал бы пустой блок без рабочего пути для оператора.

## Регрессии

- В `apps/interface/tests/test_display_actions_api.py` восстановлены и обновлены API-проверки:
  - detail API возвращает `contacts`, `photos` и `camera_link`;
  - CRUD фото через task routes остаётся рабочим;
  - monitoring не может загружать фото.
- В `frontend/src/pages/display-view/DisplayViewPage.test.tsx` добавлен сценарий с camera widget:
  - monitoring видит iframe-блок;
  - после timeout UI переключается на кнопку открытия камеры;
  - control этот блок не получает.
