# T-8-060 — Показывать все панели выбранного экрана в ЗИП

## Что исправлено

- В `frontend/src/entities/panel/hooks.ts` `usePanels()` получил опцию `fetchAll` и умеет дочитывать все cursor-страницы `/panels/`, склеивая `results` до полного массива.
- В `frontend/src/pages/zip/ZipPage.tsx` ZIP-колонки включают `fetchAll` только когда выбран конкретный экран (`displayId` есть).
- Глобальный режим `/zip` без фильтра по экрану оставлен прежним: без полного вычитывания всех панелей, чтобы не раздувать список на сценарии с тысячами записей.

## Причина бага

- ZIP-колонки читали только первую страницу `/panels/`, а panel API использует cursor pagination с лимитом 50.
- Из-за этого при выбранном экране пользователь видел обрезанный список и не мог работать с панелями, которые лежали на следующих страницах.

## Проверка

- `frontend: npm test -- src/entities/panel/hooks.test.tsx src/pages/zip/ZipPage.test.tsx src/features/panels/PanelCreateButton.test.tsx`
- `frontend: npm run typecheck`

## Регрессии

- В `frontend/src/entities/panel/hooks.test.tsx` добавлены проверки:
  - по умолчанию `usePanels()` возвращает только первую страницу;
  - при `fetchAll: true` хук дочитывает `next_cursor` и склеивает все страницы.
- В `frontend/src/pages/zip/ZipPage.test.tsx` добавлена проверка, что `ZipPage` реально включает `fetchAll` для display-filtered колонок.

## Архитектурное решение

- Backend для этой карточки не менялся.
- Выбран frontend-only путь, потому что задача локальна для экранно-отфильтрованного режима ЗИП и не требует менять общие правила пагинации panel API.
