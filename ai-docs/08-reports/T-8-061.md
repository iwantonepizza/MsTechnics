# T-8-061 — Контраст селекта экрана и формы создания панели в ЗИП

## Что исправлено

- В `frontend/src/pages/zip/ZipPage.tsx` селект выбора экрана получил более контрастные цвета:
  - текст переведён на `var(--fg)`;
  - фон и граница подняты до явных `var(--bg-1)` и `var(--border-subtle)`;
  - добавлен `data-testid="zip-display-filter"` и стили для `<option>`.
- В `frontend/src/features/panels/PanelCreateButton.tsx` форма `+ Панель` приведена к единому стилю полей:
  - labels теперь используют основной цвет текста;
  - `input`, `select`, `textarea` вынесены на общий контрастный набор стилей;
  - варианты `<option>` в селекте экрана тоже получают читаемый foreground/background.

## Причина бага

- Поля формы и селект экрана использовали приглушённый foreground и серый фон, из-за чего в обеих темах текст проседал по контрасту.
- Проблема была именно в UI-токенах и повторялась в двух местах одного потока: фильтр экрана и модалка создания панели.

## Проверка

- `frontend: npm test -- src/pages/zip/ZipPage.test.tsx src/features/panels/PanelCreateButton.test.tsx`
- `frontend: npm run typecheck`

## Регрессии

- В `frontend/src/pages/zip/ZipPage.test.tsx` добавлена проверка контрастных inline-стилей для селекта экрана.
- В `frontend/src/features/panels/PanelCreateButton.test.tsx` добавлена проверка стилей полей модалки создания панели.
