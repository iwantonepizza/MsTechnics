# T-8-080 — Мобильный layout экрана отдела без перекрытия колонок

## Что исправлено

- В `frontend/src/pages/display-view/DisplayViewPage.tsx` текущий desktop-layout переведён на mobile-first responsive-обвязку:
  - корневой контейнер теперь складывает экран в вертикальный стек и только на `lg` возвращается к 3 колонкам;
  - grid, detail и rail на мобиле занимают `w-full` и идут друг под другом;
  - для grid-колонки добавлен ограниченный по высоте scroll-контейнер, чтобы сетка не выталкивала остальные блоки за экран;
  - фиксированные ширины detail/rail сохранены только для desktop.
- В шапке экрана название сделано `truncate`, чтобы длинное имя экрана не ломало узкую ширину.

## Причина бага

- Страница была жёстко собрана под desktop: `flex-row` + фиксированные 360px и 320px справа.
- На телефоне detail и rail физически съедали ширину, а сетка уходила под них и становилась частично недоступной.

## Проверка

- `frontend: npm test -- src/pages/display-view/DisplayViewPage.test.tsx`
- `frontend: npm run typecheck`

## Регрессии

- В `frontend/src/pages/display-view/DisplayViewPage.test.tsx` добавлена проверка на responsive-обвязку:
  - root хранит `flex-col` + `lg:flex-row`;
  - grid-column остаётся нижне-границей на мобиле и правой границей на desktop;
  - detail/rail колонки имеют `w-full` на мобиле и фиксированные `lg`-ширины.
- Существующие сценарии для role matrix и camera widget прогнаны повторно на том же экране.

## Отклонения от mobile plan

- Полный `DisplayViewMobile` из `ai-docs/07-frontend/mobile-adaptation-plan.md` в этой задаче не реализовывался.
- Для `T-8-080` сделан локальный безопасный фикс текущего layout без смены user flow и без отдельного mobile-дерева компонентов.
