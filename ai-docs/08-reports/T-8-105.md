## Что сделано

- Исправлены битые русские подписи quick links в списке экранов: `ЗИП` и `Заявки`.
- Виджет ежедневных задач теперь fallback-ит с городского списка на общий список, если для текущего города задач нет.
- В темной теме исправлены базовые токены `bg-2/bg-3/bg-4`, из-за которых secondary-кнопки, hover, tabs и dropdown выглядели светлыми со светлым текстом.
- Для native `select/option` добавлены theme-aware цвета и `color-scheme`.
- Header/login логотип получил `theme-logo` и становится белым в dark/system-dark.
- В отделе `monitoring` скрыты действия физической мутации панели: смена состояния, смена отдела, снять панель, поставить панель. Создание заявки для проблемной панели осталось доступным.

## Архитектурные решения

- Daily tasks fallback оставлен на уровне frontend hook/widget без изменения API: сначала используется фильтр по городу, затем общий список при пустом результате.
- Ограничение panel mutation завязано на `department !== 'monitoring'`, а не только на роль пользователя, потому что требование сформулировано именно для отдела monitoring.
- Dark theme исправлена через design tokens, чтобы не чинить каждую кнопку и выпадающий список вручную.
