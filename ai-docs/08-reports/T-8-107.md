# T-8-107 — Prod UI stability: activity feed, cities retry, mobile camera, SSE

## Результат

Hotfix развёрнут на production 2026-06-04 через git, commit `de8c6ab`.

- Устранён бесконечный цикл запросов ActivityLog: `since` теперь стабилен между render.
- `429 Too Many Requests` больше не повторяется автоматически общим retry-механизмом.
- Лента действий показывает явную ошибку и допускает ручной повтор.
- Повтор загрузки списка экранов повторяет оба запроса: `/cities/` и `/displays/`.
- На главной возвращён фильтр ленты по периоду и виду сущности.
- Camera iframe больше не удаляется по пятисекундному таймауту; внешняя ссылка остаётся доступной.
- Native production Gunicorn переведён с sync workers на `gevent`.
- Для SSE добавлен отдельный nginx location без buffering, cache и access log.

## Корневые причины

- Activity feed вычислял ISO-время с новыми миллисекундами при каждом render и включал его в TanStack Query key.
- Общий retry повторял ответы `429` и усиливал запросный шторм.
- Sync Gunicorn workers обслуживали долгоживущий SSE и регулярно завершались по timeout через 120 секунд.
- Camera iframe заменялся fallback-блоком раньше, чем успевал загрузиться на мобильной сети.
