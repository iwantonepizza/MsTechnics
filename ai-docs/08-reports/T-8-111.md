## Архитектурные решения

- Исторические списки используют существующий cursor contract, без загрузки всех записей одним запросом.
- Media URL возвращается только после проверки physical storage.
- Очистка истории не затрагивает пользователей, заявки, экраны, панели, ячейки, уведомления и VNNOX alarms.
- Missing media очищается автоматически только по явному `--apply`; удаление заглушек требует отдельный флаг.
