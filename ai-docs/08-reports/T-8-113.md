# T-8-113. Owner feedback: mobile tasks, activity period, upload media type, active application counts

Дата: 2026-06-05
Исполнитель: GPT-5

## План

1. Обновить `ActivityFeedBand`: периоды `Сегодня`/`Всё время`, default `Сегодня`.
2. Исправить `apiClient`, чтобы `FormData` не наследовал `Content-Type: application/json`.
3. Исправить backend `application_count`, исключив архивные статусы.
4. Разрешить `control` выполнять ежедневные задачи и сделать task row настоящей кнопкой для touch.
5. Добавить мобильный быстрый блок задач отдела контроля на главную.
6. Прогнать тесты/build, запушить и развернуть.

## Выполнение

## Сделано

- `ActivityFeedBand` на главной получил переключатель периода:
  - `Сегодня` — default, отправляет `since` на начало текущего локального дня;
  - `Всё время` — не отправляет `since`.
- `apiClient` теперь нормализует headers через `AxiosHeaders` и для `FormData` ставит `Content-Type=false`, чтобы browser/axios сформировали multipart boundary сами.
- `DisplayViewSet.list` считает `application_count` только по неархивным заявкам, исключая `archive_done` и `archive_unable`.
- `DailyTasksPanel` использует настоящий `<button>` для строки задания, с touch-friendly hit target.
- Backend разрешает выполнять ежедневные задачи ролям `monitoring`, `control`, `admin`, `all`; `service` по-прежнему получает `403`.
- На главной для роли `control` добавлен мобильный блок `Задачи контроля` над основными колонками (`md:hidden`), чтобы задачи были доступны сразу после входа.
