# T-8-114. Mobile monitoring daily tasks on main page

Дата: 2026-06-06
Исполнитель: GPT-5

## Что сделано

- На главной `/menu` мобильный блок ежедневных задач переведён с control-семантики на monitoring-семантику.
- Блок оставлен в потоке страницы под KPI и скрыт на desktop через `md:hidden`, без `fixed`/`absolute`, чтобы не перекрывать остальные блоки.
- Проверка доступа учитывает legacy `permission` и профильные `roles`; пользователь с ролью `monitoring` увидит блок даже при multi-role профиле.
- Старый текст «Задачи контроля» заменён на «Ежедневные задачи» / «Быстрый доступ для мониторинга».
- Дополнительное backend/frontend логирование не добавлялось: изменение frontend-only, а backend уже пишет структурные JSON-логи через `structlog` в stdout/systemd journal.
