## 12. Починить уведомления
> «Починить уведомления (разобраться почему в апреле отвалились полностью)»

- **Статус:** 🟢 готова (root cause ясен)
- **Формальные задачи:** см. `06-integrations/telegram-russia-workaround.md` и `06-integrations/notifications-redesign.md`
  - `T-1-012-telegram-healthcheck` — добавить health-check на отправку в TG (раз в 5 минут пробует, пишет в лог/sentry)
  - `T-5-010-socks5-proxy` — прокси через зарубежный сервер
  - `T-5-011-redis-streams-migration` — переход с pubsub на Streams (гарантированная доставка)
  - `T-5-012-notification-dedup` — дедупликация в течение N минут
- **Комментарий архитектора:**
  - Root cause: Telegram API (`api.telegram.org`) частично блочится в РФ. Бот иногда работает, иногда нет. На pubsub потерянные сообщения не ретраятся.
  - Решение: прокси (MTProxy / SOCKS5) + Streams + ретраи.
