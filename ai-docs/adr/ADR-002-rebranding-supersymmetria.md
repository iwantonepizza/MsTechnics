# ADR-002. Rebranding: «MsTechnics» → «Суперсимметрия»

**Дата:** 2026-05-13
**Статус:** accepted
**Контекст:** решение владельца, раунд продуктовых ответов 2026-05-13

---

## Контекст

Владелец принял решение перебрендировать продукт с рабочего имени «MsTechnics» на «Суперсимметрия». Прислан новый логотип, новая палитра (см. `07-frontend/brand-palette-supersymmetria.md`) и явное требование на тёмную тему. Домен **не меняется** (`mstechnics.ru` или текущий — остаётся).

**Обнаружено в брендбуке (2026-05-16):** Суперсимметрия — это **DOOH-агентство** (Digital Out-of-Home). Старое имя компании — **msgroup**, откуда и пошёл `mstechnics` в коде. Брендбук рассказывает легенду: «компания msgroup делала первые шаги в наружной рекламе… имя сдерживало идею… переход к Суперсимметрии стал символом нового этапа». То есть наша система — это **внутренний инструмент агентства** Суперсимметрия для обслуживания их собственного парка LED-панелей. Это **не клиентский SaaS**, и брендбук применяется в нашем UI **сдержанно**: только палитра + шрифты + логотип + название. Стилеобразующие элементы (рамка «машинного зрения», маркерные плашки) — для маркетинговых материалов компании, не для табличных UI.

Полные brand guidelines для разработчиков — в [`07-frontend/brand-guidelines-supersymmetria.md`](../07-frontend/brand-guidelines-supersymmetria.md). PDF брендбука (115 страниц) и гайдбука (50 страниц) — в `dumps/brand-pdf/` локально, **в git не коммитим**.

---

## Решение

1. **Имя продукта в UI** — «Суперсимметрия». Все `<title>`, login page, header, footer, email-шаблоны, MAX/TG sender name.
2. **Домен и внутренние идентификаторы остаются** «mstechnics»:
   - Django app names (`apps/core`, `Config/settings.py`, …) — не переименовываем (это сломает миграции и imports).
   - Имя БД (`mstechnics`) — не трогаем.
   - URL домена — владелец не просил менять.
   - Имя репозитория — не критично, опционально позже.
3. **Логотип** — новый, SVG в `frontend/public/logo-supersymmetria.svg`. Favicon, PWA manifest, login, header — все используют его.
4. **Палитра** — новая (`brand-palette-supersymmetria.md`). Старая `tokens.css` v1 (T-4-001) **полностью заменяется** v2 (T-7-002).
5. **Тёмная тема** — добавляется как первоклассный режим. Toggle в хедере + persist в `localStorage`. Default — следовать `prefers-color-scheme`.

---

## Что НЕ затрагивается

- **Цвета состояний панелей** (`Condition.color`, `Condition.color_text`) — это данные владельца в БД, не дизайн-токены. Не трогаются.
- **Семантические алёрты** (красный для ошибок, жёлтый для предупреждений, оранжевый для «низкого остатка ЗИП») — остаются. Они вне брендинговой палитры.
- **Имена Python модулей / Django apps** — не переименовываются. Это внутренняя архитектура, не product surface.

## Последствия

### Положительные

- Чистая палитра, согласованная с брендбуком владельца.
- Dark mode (поднимется в T-7-002) — реальное требование пользователей-сервисменов работающих на разных устройствах в разное время суток.
- ADR фиксирует, что **внутренние имена остаются** — это спасает от соблазна «всё переименовать заодно».

### Отрицательные / риски

- Двойственность name'инга («Суперсимметрия» в UI, «mstechnics» в коде/БД/URL) — может быть точкой непонимания при онбординге новых разработчиков. Уменьшается описанием в AGENTS.md.
- Необходимость прогнать audit всех мест в frontend, где захардкожено «MsTechnics» (login title, README copy, error pages, email templates).
- Тёмная тема **должна** пройти WCAG-проверку контрастности — палитра выглядит контрастной (17:1 для primary), но для secondary/disabled-уровней надо проверить.

---

## Альтернативы (отброшены)

- **Полное переименование (включая код, БД, URL).** Отброшено: это месяцы работы (миграции, обновление всех клиентов, DNS, SSL-сертификаты), без существенной выгоды. Бренд живёт в UI, бэкенду имя индифферентно.
- **Оставить старую палитру, только название и логотип.** Отброшено: владелец явно прислал новые цвета, dark theme — отдельное требование.
- **Сделать палитру customizable через админ-панель.** Отброшено: овер-инжиниринг под текущий масштаб (4 активных юзера, один тенант).

## Open questions для будущих ADR

- Если масштаб вырастет (мульти-тенант, разные клиенты с разными брендами) — нужно ли вернуться к customizable themes? Пока **нет**.
- Кириллический «Суперсимметрия» в URL slug'ах (например, `/superсимметрия/login`)? — **Нет**. URL остаются latin. Имя только в UI.
